  • HTML Attributen

  • Inputs in HTML zijn elementen waarmee een gebruiker gegevens kan invoeren in een formulier. Ze worden gemaakt met het input element en kunnen verschillende types hebben, zoals tekstvelden, checkboxen, radio-buttons, knoppen, datums, wachtwoorden en meer. Volgens W3Schools zijn er 22 officiele inputs.

  • Attributen

    Voordat ik de type inputs laat zien, is het ook belangrijk om te weten wat attributen zijn, wat ze doen, en op welke types je deze kan gebruiken.

  • Globale attributen

    Globale attributen zijn HTML attributen die je op elk HTML element kunt gebruiken, dus ook op alle inputs.

  • ID attribuut

    Het id attribuut is een unieke identificatie dat je op elk HTML element kunt gebruiken. Het mag maar één keer per pagina voorkomen en wordt vooral gebruikt om elementen te koppelen aan CSS, JavaScript of een label in formulieren. Hierdoor kun je een element eenvoudig selecteren of stylen, en weet een label precies met welke input het verbonden is.

    Waardes

    Je mag zelf bepalen welke waarde je gebruikt, zolang er geen spaties in zitten, het niet met een cijfer begint, en het maar één keer voorkomt op de pagina.

    MDN: id attribute

  • Class attribuut

    Het class attribuut gebruik je om een element één of meerdere classnamen te geven, zodat je het kunt groeperen voor styling in CSS of selectie in javascript. In tegenstelling tot id mag een class meerdere keren voorkomen op dezelfde pagina en kan een element ook meerdere classes tegelijk hebben. Dit maakt het ideaal om herbruikbare stijlen toe te passen.

    Waardes

    Een class mag meerdere classnamen bevatten, gescheiden door spaties, en mag ook met een cijfer beginnen (in tegenstelling tot id). Binnen één classnaam mogen alleen geen spaties staan.

    MDN: class attribute

  • Style attribuut

    Het style attribuut gebruik je om inline CSS toe te voegen aan een HTML element. Hiermee kun je direct stijlen toepassen, zoals kleuren, marges, groottes of lettertypes. Het is handig voor snelle, kleine aanpassingen, maar minder geschikt voor grote projecten omdat stijlen dan moeilijker te onderhouden zijn.

    Waardes

    De waarde van het style attribuut is een CSS regel of meerdere CSS regels die je direct op het element zet.

    MDN: style attribute

  • Title attribuut

    Het title attribuut geeft extra informatie over een element in de vorm van een tooltip. Wanneer je met je muis over het element hovert, toont de browser deze tekst. Het wordt vaak gebruikt om verduidelijking, uitleg of hints te geven aan de gebruiker.

    Waardes

    Bevat een korte uitleg of extra informatie over een element. De waarde is gewoon tekst.

    MDN: title attribute

  • Lang attribuut

    Het lang lang attribuut geeft aan in welke taal de inhoud van een element geschreven is. Browsers, zoekmachines en schermlezers gebruiken dit om tekst correct uit te spreken of te tonen. Het wordt meestal op het HTML element gezet, maar je kunt het ook op losse elementen gebruiken wanneer een stukje tekst in een andere taal is.

    Waardes

    De waarde van het lang attribuut is altijd een ISO (International Organization for Standardization) taal code. Zoals nl en en. Je zou ook subtalen/dialecten kunnen koppelen met een streepje. Bijvoorbeeld: en-US (Engels Verenigde staten) en en-GB (Engels Verenigd Kroninkrijk). Of pt-BR (Portugees Brazilie) en pt-PT (Portugees Portugal).

    MDN: lang attribute

  • Dir attribuut

    Het dir attribuut bepaalt de textdirection van de inhoud binnen een HTML element. Dit is vooral belangrijk voor talen die rechts naar links gelezen worden, zoals Arabisch of Hebreeuws.

    Waardes

    Dir kan drie waardes hebben: ltr, rtl, en auto. Ze bepalen of tekst links naar rechts, rechts naar links, of automatisch gedetecteerd wordt.

    MDN: dir attribute

  • Hidden attribuut

    Het hidden attribuut maakt een HTML element onzichtbaar op de pagina. Het element blijft in de DOM aanwezig, maar wordt niet weergegeven en neemt geen ruimte in beslag. Het is handig voor content die je tijdelijk wilt verbergen zonder het te verwijderen uit de HTML.

    Waardes

    Het hidden attribuut is een boolean attribuut. Dat betekent dat het geen specifieke waarde nodig heeft. De aanwezigheid van het attribuut maakt het element verborgen.

    MDN: hidden attribute

  • Tabindex attribuut

    Het tabindex attribuut bepaalt de volgorde van focus wanneer een gebruiker door de pagina navigeert met de tab toets. Hiermee kun je bepalen welke elementen eerst, later of helemaal niet gefocust kunnen worden.

    Waardes

    0 maakt het element focusbaar in de normale volgorde van de pagina. Een positief getal geeft een aangepaste volgorde, waarbij elementen met een lagere waarde eerst focus krijgen. -1 zorgt ervoor dat het element niet via tab kan worden gefocust, maar het kan nog wel gefocust worden met JavaScript of een muisklik.

    MDN: tabindex

  • Draggable attribuut

    Het draggable attribuut bepaalt of een HTML element gesleept kan worden met drag-and-drop functionaliteit.

    Waardes

    Met draggable bepaal je of een element drag-and-drop interactie ondersteunt (true), of niet (false), of dat het standaardgedrag geldt (afhankelijk van het element).

    MDN: draggable

  • Contenteditable attribuut

    Het contenteditable attribuut maakt een HTML element bewerkbaar door de gebruiker. Als je dit attribuut toevoegt, kan de gebruiker de tekst in het element direct aanpassen in de browser, zonder dat er een input of textarea nodig is.

    Waardes

    Contenteditable maakt een element bewerkbaar of alleen lezen, afhankelijk van de waarde true of false.

    MDN: contenteditable

  • Spellcheck attribuut

    Het spellcheck attribuut bepaalt of de browser spelling en grammaticacontrole moet uitvoeren op de inhoud van een element. Het is vooral nuttig bij tekst velden zoals input en textarea om gebruikers te helpen fouten te vermijden.

    Waardes

    Deze kan drie vormen hebben: met spellcheck="true" staat spellingcontrole aan, met spellcheck="false" staat het uit, en wanneer je het attribuut helemaal weglaat bepaalt de browser zelf of spellcheck wordt ingeschakeld, afhankelijk van het element en de instellingen.

    MDN: spellcheck

  • Translate attribuut

    Het translate attribuut bepaalt of de inhoud van een HTML element door automatische vertaaldiensten (zoals Google Translate) vertaald mag worden. yes: de inhoud mag vertaald worden (standaard) en no: de inhoud mag niet vertaald worden.

    Waardes

    Je geeft aan of de tekst door automatische vertaaldiensten moet worden meegenomen (yes) of juist niet (no).

    MDN: translate

  • Input attributen

    Input attributen zijn speciale attributen die je aan een input element kunt meegeven om het gedrag, de invoerbeperkingen en het uiterlijk van het input te regelen. Ze maken het mogelijk om bijvoorbeeld te bepalen welk type gegevens de gebruiker kan invoeren, of een veld verplicht is, of er een standaardwaarde staat, en nog veel meer. Deze attributen werken alleen op inputs en soms op een textarea en button.

  • Type attribuut

    type Het type attribuut van een input element zegt eigenlijk wat voor soort invoerveld het is en hoe het zich gedraagt. Afhankelijk van de waarde van type weet de browser bijvoorbeeld dat het een gewoon tekstveld moet laten zien, een wachtwoordveld, een knop, of een datumkiezer etc.

    Waardes

    Tekst: text, password, search, tel, url en email.
    Numeriek: number, range, date, datetime-local, month, week en time. Keuzevelden: checkbox, radio, file en color.
    Knoppen: button, submit, reset en image. en hidden.

    MDN: type input

  • Name attribuut

    name Het name attribuut bij een input geeft het “naamlabel” van het veld aan. Dit is belangrijk wanneer je een formulier verstuurt, want de browser gebruikt de name om te weten welke waarde bij welk veld hoort.

    Waardes

    Dit kan vrij gekozen tekst bevatten, zolang het maar een geldige HTML naam is. Mag letters, cijfers, underscores(_) en streepjes(-) bevatten.

    W3Schools: name attribuut

  • Value attribuut

    value Het value attribuut van een input bepaalt wat er standaard in het beld staat of welke waarde wordt verstuurd als het formulier wordt ingediend.

    Waardes

    De waardes van value zijn altijd door jezelf gekozen tekststrings, en de inhoud hangt af van het inputtype. Het attribuut heeft dus geen vaste set opties, jij bepaalt de inhoud.

    W3Schools: value attribuut

  • Placeholder attribuut

    placeholder Het placeholder attribuut bij een input geeft een voorbeeldtekst of hint weer in een input voordat de gebruiker iets typt. Het is een visueel iets om de gebruiker te laten weten wat voor soort informatie de gebruiker in het veld moet invullen, bijvoorbeeld een naam, e-mailadres of telefoonnummer.

    Waardes

    De waarde van placeholder is altijd gewone tekst die verdwijnt zodra de gebruiker begint te typen. Het is dus geen echte ingevulde waarde, maar puur een visuele hint om het veld duidelijker te maken.

    MDN: placeholder attribuut

  • Required attribuut

    required Het required attribuut zorgt ervoor dat een input veld verplicht ingevuld moet worden voordat een formulier kan worden verzonden. Als de gebruiker probeert het formulier te versturen zonder iets in dit veld in te vullen, geeft de browser automatisch een foutmelding.

    Waardes

    De enige waarde van required is dat het aanwezig of afwezig is. Aanwezig: verplicht veld en Afwezig: niet verplicht. Soms zie je ook input required="required", Dat werkt ook, maar betekent exact hetzelfde als alleen required.

    MDN: required attribuut

  • Disabled attribuut

    disabled Het disabled attribuut maakt een input veld niet interactief. De gebruiker kan er niets in typen of selecteren, en het veld wordt vaak visueel grijs gemaakt door de browser.

    Waardes

    De waarde van dit attribuut werkt hetzelfde als het required attribuut. Als disabled aanwezig is, wordt het element uitgeschakeld; als het ontbreekt, is het element gewoon bruikbaar.

    MDN: disabled attribuut

  • Readonly attribuut

    readonly Het readonly attribuut maakt een input veld alleen lezen. De gebruiker kan de inhoud zien en eventueel selecteren, maar niet aanpassen. In tegenstelling tot disabled wordt de waarde van een readonly veld wel verzonden met het formulier.

    Waardes

    Hetzelfde als bij required en disabled, als het attribuut aanwezig is, kun je de inhoud zien maar niet aanpassen.

    MDN: readonly attribuut

  • Min / Max attribuut

    min / max De min en max attributen bij een input element stellen de minimum- en maximumwaarde in die een gebruiker mag invoeren. Ze worden vooral gebruikt bij inputtypes zoals number, range, date, datetime-local, month, time en week. Bijvoorbeeld een gebruiker mag alleen een tijd kiezen tussen 06:00 en 18:00 bij de time input.

    Waardes

    De waarde van min en max is altijd een getal of datum/tijd, afhankelijk van het type input.

    MDN: min attribuut MDN: max attribuut

  • Minlength / Maxlength attribuut

    minlength / maxlength De minlength en maxlength attributen bij een input element geven aan hoeveel tekens een gebruiker minimaal of maximaal mag invoeren. Ze worden meestal gebruikt bij tekstvelden zoals text, password, email, etc.

    Waardes

    De waardes van minlength en maxlength zijn altijd positieve getallen die het toegestane aantal tekens voor het veld bepalen.

    MDN: minlength attribuut MDN: maxlength attribuut

  • Step attribuut

    step Het step attribuut bij een input bepaalt de stapgrootte voor invoervelden met een nummer- of datumwaarde, zoals number, range, date, datetime-local, month of time. Het geeft aan hoeveel je omhoog of omlaag kunt gaan met pijltjes of schuifbalken. Het bepaalt in welke stapjes de waarde kan veranderen, zodat invoer makkelijker en consistenter wordt.

    Waardes

    De waarde van dit attribuut is een een getal of speciale waarde die bepaalt hoe groot de stappen zijn waarmee de gebruiker een waarde kan verhogen of verlagen. Je kan ook als waarde any meegeven, dit betekent dat je alle waardes zoals komma getallen kan invoeren in een input. Decimalen worden namelijk niet altijd geaccepteerd door de browser validatie. De browser gebruikt namelijk altijd standaard een stap van 1.

    MDN: step attribuut

  • Autocomplete attribuut

    autocomplete Het autocomplete attribuut bij een input vertelt de browser of het veld automatisch ingevuld mag worden met eerder ingevoerde waarden of gegevens die opgeslagen zijn. Handig bij formulieren zoals e-mail, adres of creditcard, zodat gebruikers sneller kunnen invullen.

    Waardes

    on: de browser mag automatisch suggesties geven of invullen. off: de browser mag geen automatische invulling doen.

    MDN: autocomplete attribuut

  • Autofocus attribuut

    Het autofocus attribuut zorgt ervoor dat een element, zoals een een input, automatisch geselecteerd en actief wordt zodra de pagina wordt geladen. De gebruiker kan meteen typen of interageren met dat veld zonder eerst te hoeven klikken.

    Waardes

    De aanwezigheid van het attribuut zorgt dat het element direct focus krijgt bij het laden van de pagina; afwezigheid betekent dat het geen automatische focus krijgt.

    MDN: autofocus attribuut

  • Pattern attribuut

    Het pattern attribuut bij een input bepaalt een regel waaraan de invoer moet voldoen, met behulp van een regular expression (regex). Het wordt vaak gebruikt bij tekstvelden zoals text, email of tel om invoer te valideren voordat een formulier wordt verzonden.

    Waardes

    De waarde is altijd een regex string bijvoorbeeld: [A-Za-z]{3, 10} Hier mag de gebruiker alleen letters invoeren, minimaal 3 en maximaal 10 tekens. \d{10} Hier mag de gebruiker alleen 10 cijfers invoeren.

    MDN: pattern attribuut

  • Form attribuut

    Het form attribuut linkt een input aan een specifiek formulier. Dit is handig als je een input buiten een form element wilt plaatsen maar toch wilt laten meedoen met dat formulier.

    Waardes

    De waarde is het ID van het formulier.

    MDN: form attribuut

  • Formaction attribuut

    Het formaction attribuut geeft aan waar het formulier naartoe wordt gestuurd wanneer je op een submit knop klikt. Normaal gebruikt een formulier het action attribuut van het form element voor de verzend-URL. Met formaction kun je per knop een andere bestemming instellen.

    Waardes

    De waarde van dit attribuut is altijd een URL.

    W3Schools: formaction attribuut

  • Enctype en Formenctype attribuut

    Het enctype attribuut van een form bepaalt hoe de gegevens van het formulier worden gecodeerd wanneer ze naar de server worden verzonden. Het wordt gebruikt wanneer het formulier een method="post" heeft. Dit is vooral belangrijk bij invoervelden zoals bestanden of wanneer je speciale data wilt versturen.


    Het formenctype attribuut doet hetzelfde als enctype, maar dan specifiek voor een enkele submit knop. Dit is handig als een formulier meerdere submit knoppen heeft die elk de gegevens naar een andere bestemming of op een andere manier moeten verzenden.

    Waardes

    application/x-www-form-urlencoded (standaard). Dit is de standaard manier om formuliergegevens te versturen. Alles wat je invult, wordt omgezet naar tekst en netjes als key=waarde naar de server gestuurd. Bijvoorbeeld: als je je naam invult als Jamie Vos, wordt het verzonden als naam=Jamie%20Vos. Dit gebruik je voor gewone tekstvelden, bijvoorbeeld naam, e-mail of wachtwoord.


    multipart/form-data. Dit is nodig als je bestanden wilt uploaden, zoals een foto of document. Het zorgt dat de browser alle stukjes van het formulier apart verstuurt, zodat bestanden niet kapot gaan. Gebruik: altijd bij input types zoals file.


    text/plain. Dit verstuurt alles als platte tekst, precies zoals je het typt. Er wordt niets gecodeerd of speciaal verwerkt.

    W3Schools: enctype

  • Formmethod attribuut

    Het formmethod attribuut bepaalt op welke manier een formulier zijn gegevens naar de server verstuurt wanneer je op een submit knop klikt. Je kunt dit attribuut gebruiken op een form zelf, maar ook op een losse submit knop om de manier van verzenden voor alleen die knop te veranderen.

    Waardes

    get: De gegevens worden in de URL gezet, dus je ziet ze achter het vraagteken komen (bijv. ?naam=Jamie). Dit is handig voor zoekfuncties of filters, maar niet geschikt voor gevoelige informatie. post: De gegevens worden onzichtbaar verstuurd in de body van het verzoek. Dit is geschikter voor wachtwoorden, inlogformulieren of het uploaden van bestanden.

    W3Schools: formmethod attribuut

  • Formnovalidate attribuut

    Het formnovalidate attribuut zorgt ervoor dat de browser geen formulier validatie uitvoert wanneer je op een specifieke submit knop klikt. Normaal controleert de browser bijvoorbeeld of required velden zijn ingevuld, of een e-mailadres wel een echt e-mailadres is, enzovoort. Met formnovalidate wordt deze controle overgeslagen.

    Waardes

    Het attribuut heeft geen waarde, alleen aanwezig is al genoeg.

    W3Schools: formnovalidate attribuut

  • Formtarget attribuut

    Het formtarget attribuut bepaalt waar het resultaat van een formulier geopend wordt nadat je op de submit knop drukt. Normaal opent een formulier de nieuwe pagina gewoon in hetzelfde venster/tabblad. Maar met formtarget kun je zeggen: "Open het resultaat ergens anders."

    Waardes

    _self, (standaard) alles opent in dezelfde tab. Dit gebeurt als je niets invult. Gewoon zoals normaal dus. _blank, opent de pagina in een nieuw tabblad. _parent, opent in het bovenliggende frame, als je met frames werkt. _top, opent in het hele scherm, dus het doorbreekt alle frames. Een eigen naam (bijv. "mijnFrame") Resultaat opent in een specifiek frame met die naam.

    W3Schools: formtarget attribuut

  • Inputmode attribuut

    Het inputmode attribuut geeft aan welk soort toetsenbord of invoermethode de browser moet tonen wanneer iemand iets intypt. Dit attribuut verandert niet wat het inputveld accepteert, maar helpt alleen de gebruiker door een passend toetsenbord te tonen (vooral op mobiel). sHet is puur gebruikersvriendelijkheid, geen echte validatie.

    Waardes

    Mogelijke waardes: 1. none, Laat geen speciaal toetsenbord zien. Gebruiker moet zelf het juiste kiezen. 2. text, normaal tekst toetsenbord. 3. decimal, toetsenbord met cijfers en een decimaal teken (bijv. punt of komma). 4. numeric, cijfer toetsenbord zonder decimalen (alleen 0–9). 5. tel, telefoon toetsenbord met cijfers en telefoontekens. 6. email, Toetsenbord met @, .com en speciale e-mailtekens. 7. url, Toetsenbord met “/”, “.” en andere URL symbolen.

    MDN: inputmode attribuut

  • List attribuut

    Het list attribuut wordt gebruikt op een input om aan te geven dat het veld suggesties moet tonen uit een datalist. Het maakt dus een input met een soort “dropdown” met opties waaruit de gebruiker kan kiezen, maar de gebruiker mag ook iets anders typen.

    Waardes

    De waarde is de id van de datalist waar de input mee moet koppelen.

    W3schools: list attribuut
